### P1-B — `/api/signals/open` served paid signals unauthenticated → GATED
- **Where:** `src/app/api/signals/open/route.ts:8`
- **Problem:** `GET` ran `dbQuery` with **no auth gate** and returned up to 500 `signal_events`
  rows incl. `grade`, `ticker`, `strike`, `expiry`, `option_type`, `entry_mark`,
  `confluence_score` — i.e. the paid SPX_SLAYER + NIGHT_HAWK signal output. Live-verified
  this run: `GET /api/signals/open` → **HTTP 200** `{"ok":true,"signals":[]}` (empty only
  because market is closed; during RTH this leaks live paid signals to any anonymous caller).
- **Confirmed safe to lock down:** grepped entire `src/` — the only references to
  `signals/open` are inside the route file itself (a comment + an error log). No in-repo
  consumer fetches it, and the `signal-outcome-tracker` cron its comment cites **does not
  exist** → the route is orphaned. Gating it breaks nothing.
- **Fix applied:** added `if (!isCronAuthorized(req)) return 401` at the top of `GET`,
  mirroring the sibling signal *write* routes. `tsc --noEmit` = 0 errors after the edit.
- **Deploys on this commit** (Railway auto-deploy). Post-deploy expectation: anonymous
  `GET /api/signals/open` → **401**.
