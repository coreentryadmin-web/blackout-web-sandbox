## P2 — Fix This Week
- **options-socket shard 0 in a persistent code=1006 reconnect loop (`failures=531`).**
  Live `blackout-web` logs show, every 60s:
  `[options-socket] shard 0 connected (1 contracts)` → `reconnect in 60000ms (code=1006, failures=531)`.
  `consecutiveFailures` only resets on a successful **auth** (`src/lib/ws/options-socket.ts:405-406`),
  so 531 means the shard has not sustained an authenticated stream in ~8+ hours.
  Closes are **server-initiated 1006** (abnormal), not the stall watchdog — the watchdog
  already gates on options-RTH (`options-socket.ts:453-457`), so its off-hours suppression
  does not cover server-side drops.
  - **Impact now:** benign — markets are closed, no held contract needs a live NBBO. But it
    burns the single Massive `/options` WS slot (`MAX_CONNECTIONS=1`, `options-socket.ts:70-73`)
    and emits continuous log noise.
  - **Risk:** the unbounded counter means you cannot distinguish "531 from harmless off-hours
    churn" from "531 from real RTH breakage" — it masks a genuine RTH failure if one occurs.
  - **ACTION — verify at RTH:** re-check after 09:30 ET. If `failures` resets toward 0 once
    quotes flow, this is cosmetic off-hours churn. If it stays pinned at 60s/1006 during RTH,
    Night's Watch live position valuations are degraded → promote to P1.
  - **Suggested fix (P3):** gate `scheduleReconnect`/heartbeat on options-RTH the same way the
    stall watchdog already is, so the socket parks off-hours instead of churning and the failure
    counter reflects only real failures.
