## 1. `npm audit` — 2 moderate → FIXED (pushed to main)

**Finding:** `npm audit` reported 2 moderate, both the same root advisory **GHSA-qx2v-qp2m-jg93** (PostCSS XSS via unescaped `</style>` in CSS Stringify Output, vulnerable range `<8.5.10`).

- Root cause: `next@15.5.19` bundled its own `postcss@8.4.31` at `node_modules/next/node_modules/postcss`, which is `< 8.5.10` → vulnerable. The **top-level** `postcss` was already `8.5.15` (safe).
- Exploitability in our context: **low** — postcss runs at BUILD time over our own first-party CSS; the `</style>` stringify XSS only matters when untrusted CSS is processed and emitted. No runtime exposure. Severity moderate, build-toolchain only.
- npm's `fixAvailable` was a **nonsense major downgrade** to `next@9.3.3` — rejected (never auto-bump/downgrade majors).

**Fix (trivial-safe, build-gated → `main`):** added an npm `overrides` block pinning postcss to the already-installed patched line, deduping next's nested copy up to `8.5.15`:
```jsonc
"overrides": { "postcss": "$postcss" },   // references the direct devDep
"devDependencies": { "postcss": "^8.5.10", ... }
```
This is a same-major **minor** bump (8.4.31 → 8.5.15) of a build-time tool, fully reversible.

**Validation:** `npm install` removed the duplicate (`removed 1 package`) · `npm audit` → **found 0 vulnerabilities** · `npx tsc --noEmit` green · `npm run build` green (CSS pipeline unaffected). Nested `next/postcss` deduped away; top-level `8.5.15`.
